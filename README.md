# Ego-UniRAG

> **Важно:** Данный проект является форком репозитория [MrGAN12009/prompt-5_8](https://github.com) и развивается как самостоятельное решение под кодовым именем **Ego-UniRAG**.

## Описание проекта

**Ego-UniRAG** — это универсальная, гибко настраиваемая RAG-система (Retrieval-Augmented Generation), ориентированная на создание персонального и прикладного контекста для моделей искусственного интеллекта.

### Ключевые особенности и дополнения:

1. **Универсальность пайплайна**: Архитектура проекта спроектирована как сквозной универсальный конвейер. Это позволяет быстро разворачивать RAG-системы под самые разные прикладные задачи с полным контролем над каждым этапом: от парсинга данных до генерации финального ответа.

2. **Интеграция с ProxiAPI**: Внедрена поддержка российского провайдера ProxiAPI. Это обеспечивает стабильный, быстрый и независимый доступ к зарубежным и локальным языковым моделям (LLM) без необходимости настройки сложных путей обхода и VPN.

3. **Глубокий контроль контекста**: Возможность тонкой настройки фильтрации, ранжирования документов и управления памятью (эго-контекстом) пользователя.

4. **Терминальный интерфейс**: В текущей версии используется консольный (терминальный) интерфейс для взаимодействия с пользователем. Обмен вопросами и ответами происходит непосредственно в командной строке.

## Архитектура проекта

Проект реализует классическую RAG-архитектуру с двумя режимами работы:

```
┌─────────────────────────────────────────────────────────────────┐
│                        Ego-UniRAG                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐   │
│  │   Query      │ ──→ │   Cache      │ ──→ │ Vector Store │   │
│  │   Input      │     │   (SQLite)   │     │ (ChromaDB)   │   │
│  └──────────────┘     └──────────────┘     └──────────────┘   │
│                                                   │             │
│                                                   ↓             │
│  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐   │
│  │   Response   │ ←── │   LLM        │ ←── │  Prompt      │   │
│  │   Output     │     │  (GigaChat/  │     │  Builder     │   │
│  └──────────────┘     │   OpenAI/    │     └──────────────┘   │
│                       │   ProxiAPI)  │                         │
│                       └──────────────┘                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Режимы работы:

| Режим | Описание | LLM Provider |
|-------|----------|--------------|
| **GigaChat Mode** | Работа через API Сбера | GigaChat |
| **API Mode** | Работа через OpenAI API | OpenAI (GPT) |
| **ProxiAPI Mode** | Работа через ProxiAPI (РФ) | ProxiAPI (GPT/Claude) |

## Требования

- **Python**: 3.11+ (рекомендуется 3.13)
- **Операционная система**: Windows / Linux / macOS
- **Память**: Минимум 2 GB RAM
- **Интернет**: Для работы с внешними API

### Внешние зависимости:

- **Для GigaChat режима**:
  - Аккаунт на платформе Сбера
  - `GIGACHAT_AUTH_KEY` — ключ авторизации
  - `GIGACHAT_RQUID` — уникальный идентификатор запроса

- **Для API режима**:
  - Ключ OpenAI API
  - `OPENAI_API_KEY` — API ключ

- **Для ProxiAPI режима**:
  - Доступ к ProxiAPI прокси-серверу
  - `PROXI_API_URL` — URL прокси-сервера
  - `PROXI_API_KEY` — API ключ (опционально)

## Компоненты системы

### RAG Pipeline

Основной модуль, управляющий потоком обработки запросов:

```
1. Проверка кеша → 2. Векторный поиск → 3. Формирование промпта → 
4. Генерация ответа → 5. Сохранение в кеш
```

**Файлы:**
- `assistant_giga/rag_pipeline.py` — GigaChat режим
- `assistant_api/rag_pipeline.py` — API режим
- `assistant_proxy_api/rag_pipeline.py` — ProxiAPI режим

### Vector Store

Векторное хранилище на основе **ChromaDB** с умным чанкингом:

- Разбиение текста по абзацам и предложениям
- Перекрывание чанков (overlap) для сохранения контекста
- Использование Cosine similarity для поиска

**Файлы:**
- `assistant_giga/vector_store.py`
- `assistant_api/vector_store.py`
- `assistant_proxy_api/vector_store.py`

### Cache System

Кеширование ответов на основе **SQLite**:

- Хэширование запросов для быстрого поиска
- Хранение контекста и метаданных
- Статистика использования кеша

**Файлы:**
- `assistant_giga/cache.py`
- `assistant_api/cache.py`
- `assistant_proxy_api/cache.py`

### LLM Clients

Клиенты для работы с различными LLM:

- `GigaChatClient` — работа с API Сбера (авторизация, OAuth, embeddings)
- `OpenAI` — стандартный клиент для работы с GPT
- `ProxyAPIClient` — работа с ProxiAPI (поддержка GPT/Claude через российский прокси)

## Оценка качества

Проект включает модуль оценки качества RAG-системы через **RAGAS**.

### Запуск оценки через ProxiAPI (РЕКОМЕНДУЕТСЯ):

```bash
cd assistant_proxy_api
python evaluate_ragas.py
```

### Требования для оценки:

| Переменная | Описание | Обязательно |
|------------|----------|-------------|
| `PROXI_API_URL` | URL ProxiAPI для RAG pipeline | ✅ Да |
| `PROXI_API_KEY` | API ключ ProxiAPI | ⚠️ Опционально |
| `OPENAI_API_KEY` | API ключ OpenAI для RAGAS метрик | ✅ Да |

### Метрики RAGAS:

| Метрика | Описание | Диапазон |
|---------|----------|----------|
| **Faithfulness** | Соответствие ответа предоставленному контексту | 0.0 - 1.0 |
| **Context Precision** | Качество извлечённого контекста для ответа | 0.0 - 1.0 |

### Настройка для оценки:

```bash
# В файле .env добавьте:
PROXI_API_URL=https://api.proxyapi.ru/openai/v1
PROXI_API_KEY=your_proxi_api_key
OPENAI_API_KEY=your_openai_api_key  # Требуется для RAGAS метрик
```

### Подготовка датасета:

Файл `evaluate_ragas.py` содержит список тестовых вопросов. Для кастомной оценки отредактируйте константу `EVALUATION_QUESTIONS`:

```python
EVALUATION_QUESTIONS = [
    "Вопрос 1?",
    "Вопрос 2?",
    # ... добавьте ваши вопросы
]
```

## Пример использования

### Базовый запрос:

```python
from rag_pipeline import RAGPipeline

# Инициализация
pipeline = RAGPipeline(
    collection_name="my_collection",
    cache_db_path="cache.db",
    data_file="data/docs.txt",
    model="gpt-4o-mini"
)

# Отправка запроса
result = pipeline.query("Что такое машинное обучение?")

# Получение результата
print(result["answer"])
print(f"Из кеша: {result['from_cache']}")
```

### Получение статистики:

```python
stats = pipeline.get_stats()
print(f"Документов в хранилище: {stats['vector_store']['count']}")
print(f"Записей в кеше: {stats['cache']['total_entries']}")
```
